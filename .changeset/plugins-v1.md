Introduce the Kimi Code plugin protocol rooted at `plugin.json`, with `.kimi-plugin/plugin.json` for multi-harness repositories such as Superpowers. Plugins can contribute skills, command-backed tools, declarative `sessionStart.skill`, `skillInstructions`, display metadata, and opt-in `mcpServers`. Plugin installs copy local paths and Zip URLs into Kimi Code's managed plugin directory while keeping the original source as display metadata; reinstalling the same plugin id overwrites the managed copy while preserving enabled state and plugin MCP toggles. Plugin changes are picked up by new sessions; `/plugins reload` refreshes plugin records and diagnostics but does not hot-reload the current session. The `/plugins` command now opens an interactive manager for install, details, enable/disable, confirmed removal, reload, marketplace installs and updates, and plugin MCP toggles, while shortcut subcommands remain available. The interactive manager supports arrow-key navigation, `Space` shortcuts for install/update and enable/disable actions, and clearer marketplace install/update progress and success feedback. Plugin tools are resolved inside the managed plugin root, exposed as `plugin__<plugin>__<tool>`, and run only when called through the normal tool approval flow. Plugin MCP servers are parsed and shown at install time but only start after the user explicitly enables a server from `/plugins` or with `/plugins mcp enable` and starts a new session. Executable hooks, legacy `config_file`/`inject` adapters, `.codex-plugin/plugin.json` fallback loading, and hard-coded Superpowers behavior are not enabled by the plugin loader.
